# CVE Fix Agent

You are a specialist agent fixing a known CVE in a dependency.

## Your Task

1. **Setup branch**
   ```bash
   python3 .claude/skills/security-engineer/run_agent.py git-setup <alert_id>
   ```
   If output is `branch_exists_locally`, output `SKIPPED: branch already exists` and stop.

2. **Read the dependency manifest** ‚Äî `source` will point to a file like `go.mod`, `package.json`, or `requirements.txt`. Use Read tool.

3. **Apply the fix** ‚Äî bump the vulnerable package to the patched version from `recommendation`.

4. **Run the package manager** to regenerate the lockfile:
   - Go: `cd <module_dir> && go mod tidy`
   - Node: `cd <package_dir> && npm install` (or `yarn install`)
   - Python: `pip-compile` or `pip install -r requirements.txt --upgrade-package <pkg>`

5. **Verify** ‚Äî Read the manifest again to confirm the version was bumped.

6. **Commit** (includes both manifest and lockfile via `git add -A`)
   ```bash
   python3 .claude/skills/security-engineer/run_agent.py git-commit <alert_id> "fix(security): bump <package> to patched version (<alert_id>)"
   ```

7. **Open PR** with the template below.

8. **Return to main** ‚Äî `git checkout main`

9. **Output** PR URL or `FAILED: <reason>`.

## Fix Patterns

### Go ‚Äî `go.mod` version bump

```
# Before
require golang.org/x/net v0.0.0-20210119194325-5f4716e94777

# After ‚Äî use the patched version from the alert's recommendation
require golang.org/x/net v0.17.0
```

After editing `go.mod`, run in the module directory:
```bash
go mod tidy
```
This regenerates `go.sum`. Both files must be committed.

If `go mod tidy` fails (network issue, proxy unavailable), still commit the `go.mod` change and note in the PR body that `go mod tidy` must be run locally before merging.

### Node.js ‚Äî `package.json` version bump

```json
// Before
"dependencies": {
  "lodash": "4.17.4"
}

// After
"dependencies": {
  "lodash": "4.17.21"
}
```

Then run `npm install` or `npm audit fix` to regenerate `package-lock.json`.

### Python ‚Äî `requirements.txt`

```
# Before
requests==2.28.0

# After
requests==2.31.0
```

## Finding the Patched Version

The patched version should be in the alert's `recommendation` field. If it's not explicit:
1. Check the CVE IDs in `labels` (e.g. `CVE-2023-44487`)
2. The recommendation usually states the minimum safe version
3. Use that exact version or the latest patch in the same minor series

Do NOT bump to a major version without confirming compatibility ‚Äî prefer the minimum patched version.
